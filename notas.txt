Metodos de los arreglos:
    map() -> Iteración que manipula todo el arreglo el cual retonará un nuevo arreglo
    filter() -> Iteración que filtra el contenido de un arreglo bajo una condición retonando un nuevo arreglo
    find() -> Iteración que busca un único elemeto dentro de un arreglo y retoma ese elemento
    som() -> Iteración que busca un único elemento dentro de un arreglo y retoma true o false
    push() -> Metodo que agrega un elemento a la ultima posición del arreglo actual
    pop() -> Metodo que elimina la ultima posición de un arreglo
    shift() -> Metodo que elimina la primera posición de un arreglo
    unshift() -> Metodo que agrega un elemento a la primera posición de un arreglo
    splice() -> Metodo que elimina un elemento de una posición n del arreglo (n puede ser cualquier valor)
    forEach() -> Lo mismo que el map pero sin retorno de arreglo

    Metodos Mutables -> Modificia el arreglo original

    Metodos no Mutables -> No modifica el arreglo original

El DOM  es manipulado por medio de Eventos
Eventos de usuario -> Toda acción que realiza un usuario dentro de la aplicación;
click, doble click, scroll, presionar una tecla, etc.

Eventos de documento -> Toda acción que se realiza por debajo y no es visible para el usuario.

Los eventos son funciones o acciones que se pueden definir desde el HTML o el JavaScript, y siempre seran
ejecutados por medio de funciones

Funciones -> Bloque de código que encapsulado que se puede llamar donde quiera y ejecutará 
una acción definida.
Toda función trabaja en dos tiempos: Su definición y su llamado o invocación

    Funciones regulares
        Declarativas -> Se almacenan en el alcance global de la aplicación (hoisting)
        De expresión
    Funcones Flecha 
        De expresión
    Funciones anónimas - Es una función que np tiene nombre y se ejecuta solamente donde se definió

    No retorno - Ejecuta y muestra en la misma función (void)
    Retorno - Una ejecución que hace una función y que se podrá utilizar después
